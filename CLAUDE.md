# Chat-based Pricing Table Builder 프로젝트

## Klaim 플랫폼 컨텍스트
- **Klaim**: AI 스타트업용 토큰 기반 결제 플랫폼 (getklaim.com)
- **내 역할**: Klaim 플랫폼 내 기능 개발자
- **플랫폼 기능**: Stand-alone Page, Embedded Widget 프로모션 페이지 생성

## 현재 개발 중인 기능
- **목적**: Klaim의 프로모션 페이지 생성 UI 개선
- **범위**: 기존 폼 기반 → AI 대화 기반 인터페이스 추가
- **지원 방식**: Stand-alone/Embedded 방식 모두 지원
- **위치**: Klaim 생태계 내의 기능 개선 프로젝트

## 프로젝트 상태
- 기술 설계 초안 존재 (docs/CURRENT_DASHBOARD.md)
- Klaim 서비스 문서 생성됨 (docs/KLAIM_SERVICE.md)
- 실제 구현은 아직 시작하지 않음
- 기술 스택 미정

## 할루시네이션 방지 원칙

### 탐색 우선
- 질문 전에 `Read`, `Grep`, `Glob`으로 코드베이스 먼저 확인
- 기존 패턴에서 답을 찾을 수 있으면 따르기

### 존재 확인 필수
- 라이브러리: import 전에 package.json / requirements.txt 확인
- API/함수: 호출 전에 실제 존재 여부 확인
- 파일 경로: 가정하지 말고 실제 구조 확인

### 질문 기준
- 탐색으로 해결 → 질문 없이 진행
- 탐색해도 불명확한 비즈니스 스펙 → 질문
- 새로운 API/스키마/외부 연동 → 반드시 질문

### 금지사항
- 존재하지 않는 라이브러리/API 사용
- 코드 확인 없이 "아마 이럴 것이다" 가정
- 한 번에 10개씩 질문 (핵심만)

## 날짜 및 시간 참조
- **환경 정보 확인**: <env> 태그의 "Today's date" 항상 확인
- **웹 검색 시**: 환경의 현재 연도 기준으로 검색 (2024년 아님!)
- **문서 작성 시**: 환경의 현재 날짜 사용
- **최신 정보 참조 시**: 환경 날짜 기준으로 "최신" 판단

## Claude Code 작업 방식
1. **사전 조사**: 코드 작성 전 관련 파일 읽기
2. **계획 수립**: TodoWrite 도구로 작업 계획
3. **점진적 구현**: 200줄 이하 단위로 구현
4. **검증**: 각 단계마다 테스트 실행
